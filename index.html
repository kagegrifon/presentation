JSDoc: Generating API Documentation

Why do you need to document JavaScript projects?

When look in source code

JSDoc is a way to write comments to describe the code

*/

  return first + second;

Examples of using

/**
 * Create instance of Circle.
 *
 * @constructor
 * @this  {Circle}
 * @param {number} r - Radius of circle.
 */
function Circle(r) {
  /** @private */
  this.radius = r;
  
  /** @private */
  this.circumference = 2 * Math.PI * r;
}

Examples of using

/**
  * calculate Circle length
  *
  * @deprecated
  * @this   {Circle}
  * @return {number} Length of circle.
  */
Circle.prototype.calculateCircumference = function () {
  return 2 * Math.PI * this.radius;
};

Examples of using

/**
  * Present Circle in String.
  *
  * @override
  * @this   {Circle}
  * @return {string} Infomation of Circle.
  */
Circle.prototype.toString = function () {
  return "A Circle object with radius of " + this.radius + ".";
};

Basic tags

@param (synonyms: @arg, @argument) Document the parameter to a function.
@returns(synonyms: @return) Document the return value of a function.
@deprecated Document that this is no longer the preferred way.
@example Provide an example of how to use a documented item.
@default (synonyms: @defaultvalue) Document the default value.
@class (synonyms: @constructor) This function is intended to be called with the "new" keyword.
@private This symbol is meant to be private.
@public This symbol is meant to be public.
@version Documents the version number of an item.

More information https://code-examples.net/en/docs/jsdoc/index
